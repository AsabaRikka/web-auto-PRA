# PRD - Python网页自动化录制工具（类影刀）

## 一、项目概述

**项目名称**：WebAuto Recorder
**项目类型**：RPA（机器人流程自动化）桌面应用
**核心功能**：通过可视化录制方式，自动记录用户在网页上的操作行为，并支持回放执行，实现网页操作自动化。
**目标用户**：需要进行网页自动化操作的用户，如测试人员、数据采集人员、运营人员等。

---

## 二、功能需求详述

### 2.1 核心功能模块

| 功能模块 | 描述 | 优先级 |
|---------|------|--------|
| **浏览器录制** | 内嵌Chromium浏览器，捕获用户点击、输入等操作 | P0 |
| **元素识别与记录** | 自动识别被操作元素的XPath/CSS Selector，生成描述性标签 | P0 |
| **操作步骤编辑** | 以自然语言展示操作步骤，支持编辑、删除、插入步骤 | P0 |
| **流程回放执行** | 按顺序还原录制操作，支持单步执行和连续执行 | P0 |
| **登录状态管理** | 保存Cookies/LocalStorage，支持多账号切换 | P1 |
| **智能等待设置** | 每个步骤可设置停留时间，支持条件等待 | P1 |
| **异常处理** | 元素定位失败时提示用户，支持跳过/重试/修改定位符 | P1 |
| **人类行为模拟** | 随机延迟、鼠标轨迹模拟、滚动行为模拟 | P2 |

### 2.2 用户操作流程

```
启动应用 → 创建/打开项目 → 开始录制 → 执行网页操作 → 停止录制
                                ↓
                    编辑操作步骤（可选）→ 设置等待时间 → 保存流程
                                ↓
                    执行流程 → 查看执行日志 → 异常处理 → 完成/重试
```

### 2.3 操作信息栏设计

| 步骤 | 操作类型 | 元素描述 | XPath | 状态 | 等待时间 |
|------|---------|---------|-------|------|---------|
| 1 | 点击 | "登录按钮" | //button[@id='login'] | ✅ | 2s |
| 2 | 输入 | "用户名输入框" | //input[@name='username'] | ✅ | 1s |
| 3 | 点击 | "提交按钮" | //form/button[1] | ⏳ | - |

## 三、非功能需求

### 3.1 性能需求
- 页面加载后元素识别时间 < 500ms
- 操作回放执行速度可调节（0.5x - 2x）
- 支持最多500个操作步骤

### 3.2 兼容性需求
- 支持主流网站（支持动态加载元素的网站）
- 兼容Chrome扩展

### 3.3 安全需求
- 登录状态加密存储
- 不上传用户数据到外部服务器

## 四、技术栈建议

### 4.1 技术选型对比

| 层级 | 推荐方案 | 备选方案 | 说明 |
|-----|---------|---------|------|
| **桌面框架** | **Tauri 2.0** | Electron | 轻量、安全、原生性能好，支持Python后端 |
| **前端** | **React + TypeScript** | Vue3 | 成熟生态，组件丰富 |
| **浏览器内核** | **Playwright (Chromium)** | Selenium | 更现代，支持Shadow DOM，元素定位精确 |
| **Python运行时** | **PyO3 + Python Odessa** | CPython | Tauri 2.0官方推荐，性能好 |
| **后端/自动化** | **Playwright Python** | - | 核心自动化库 |
| **状态存储** | **SQLite + encryption** | JSON文件 | 本地存储，安全可靠 |
| **UI组件库** | **shadcn/ui** | Ant Design | 现代、轻量、易定制 |
| **图标** | **Lucide React** | - | 开源、持续更新 |

### 4.2 推荐技术栈详解

```
┌─────────────────────────────────────────────────────────────┐
│                      Tauri 2.0 桌面应用                       │
├───────────────────────┬─────────────────────────────────────┤
│      前端 (Web View)   │         后端 (Python)                │
│  ┌─────────────────┐   │   ┌─────────────────────────────┐   │
│  │ React 18        │   │   │ Playwright (浏览器自动化)     │   │
│  │ TypeScript      │   │   │ - 元素定位与操作              │   │
│  │ shadcn/ui       │   │   │ - 截图与视觉检测              │   │
│  │ Zustand (状态)   │   │   │ - 人类行为模拟               │   │
│  │ Tailwind CSS    │   │   ├─────────────────────────────┤   │
│  └────────┬────────┘   │   │ SQLite (流程/状态存储)         │   │
│           │           │   │ - 加密存储登录状态             │   │
│           │ IPC       │   │ - 项目配置管理                 │   │
│  ┌────────┴────────┐   │   └─────────────────────────────┘   │
│  │ Rust (Tauri核心) │   │                                      │
│  └─────────────────┘   │                                      │
└───────────────────────┴─────────────────────────────────────┘
```

### 4.3 关键技术点

| 技术点 | 方案 | 理由 |
|-------|------|------|
| **元素定位** | XPath为主 + CSS Selector备选 | XPath支持模糊匹配，更灵活 |
| **人类行为模拟** | Mouse弯曲轨迹 + 随机延迟 | 降低被反爬虫检测概率 |
| **登录状态保存** | BrowserContext持久化 | Playwright原生支持 |
| **等待机制** | 智能等待 + 固定等待 | Dynamic wait + explicit wait |

## 六、核心问题与解决方案

| 问题 | 解决方案 |
|-----|---------|
| 动态元素定位失败 | 使用多定位符策略（XPath + 文本内容 + 邻近元素） |
| 防止被检测为机器人 | 随机延迟 + 鼠标轨迹 + 真实视口 |
| 页面加载慢导致失败 | 智能等待 + 重试机制 + 超时配置 |
| 复杂页面元素定位不准 | 支持手动校正 + 可视化选择器 |
| 登录状态跨会话保持 | BrowserContext持久化 + 加密存储 |
